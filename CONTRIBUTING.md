# Contributing to Snap-A-Steg

Thank you for your interest in contributing! Snap-A-Steg is open source, and we welcome help with code, documentation, and testing.

## Join the Community:

We coordinate development, testing, and support on Discord
**Join the [Snap-A-Steg Discord](https://discord.gg/m9rUbBbHKR)**

### 1. Fork and Clone
- Fork the repository on GitHub.
- Clone your fork locally:
    git clone https://github.com/<your-username>/snap-a-steg.git

### 2. Install Dependencies
- Make sure you have Python 3.8+ and the requirements installed:
    pip install -r requirements.txt

### 3. Run the App
- Ensure everything works before making changes:
    python snap_a_steg.py

### 4. Make Changes
- Use clear, descriptive commit messages
- Stick to the existing code style and formatting
- Add comments if your code is complex

### 5. Submit a Pull Request
- Push your branch to your fork
- Open a pull request to the main branch of the main repo
- Reference related issues in your pull request

### Areas Where Help is Needed
- Cross-platform testing and packaging: ensure Snap-A-Steg works smoothly on macOS and Linux
- GUI improvements: usability, styling, accessibility
- Documentation: clarify instructions, fix typos, add examples
- Testing: encode/decode reliability, edge cases
- Features: CLI mode, drag-and-drop, better error handling


### Notes
- Small contributions are welcome!
- You do not need to be an encryption expert to help
- Be patient and respectful. We'll review and provide feedback
- Don't worry if it's your first open-source contribution, even small fixes are greatly appreciated!
